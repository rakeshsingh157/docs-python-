üóÑÔ∏è Database Design

Schema Overview

MySQL database with 4 core tables optimized for performance and data integrity.

users

User authentication and profile data
user_id VARCHAR(36) PK UUID primary key
username VARCHAR(50) UNIQUE Unique username
email VARCHAR(100) UNIQUE Login credential
password_hash VARCHAR(255) Bcrypt hashed password

events

Tasks and events storage with completion tracking
id INT AUTO_INCREMENT PK Event identifier
user_id VARCHAR(36) FK References users.user_id
title VARCHAR(255) Event title
date DATE Event date (YYYY-MM-DD)
time TIME Event time (HH:MM:SS)
done BOOLEAN Completion status

collaborations

User partnerships and team invitations
inviter_id VARCHAR(36) FK User who sent invitation
invitee_id VARCHAR(36) FK User who received invitation
status ENUM pending/accepted/rejected

assigned_tasks

Task assignments between collaborators
assigner_id VARCHAR(36) FK User who assigned task
assignee_id VARCHAR(36) FK User who received task
event_id INT FK References events.id

Performance Optimization

Strategic Indexing

  • idx_user_date (user_id, date)
  • idx_user_done (user_id, done)
  • idx_invitee_status (invitee_id, status)

Query Optimization

  • Parameterized queries only
  • Efficient JOIN operations
  • User-scoped data isolation
