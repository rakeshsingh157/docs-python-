API Documentation

Welcome to the remindeme API documentation. This comprehensive guide will help you integrate with our event management system using RESTful APIs.

Base URL

http://127.0.0.1:5000

All API requests should be made to this base URL.

Quick Start

To get started with the remindeme API, you'll need to authenticate your requests and understand our response format.

Example Request 
curl -X GET "http://127.0.0.1:5000/events" \
  -H "Content-Type: application/json"

Authentication

The remindeme API uses session-based authentication for web applications and token-based authentication for API access.

Session Authentication

For web applications, use the login endpoint to establish a session:

POST /auth/login

Request Body

{
  "email": "user@example.com",
  "password": "securepassword"
}

Response

{
  "success": true,
  "message": "Login successful",
  "user": {
    "id": 1,
    "username": "johndoe",
    "email": "user@example.com"
  }
}

Registration

POST /auth/register

Request Body

{
  "username": "johndoe",
  "email": "user@example.com",
  "password": "securepassword"
}

Users

Manage user profiles and account information.

Get User Profile

GET /user/profile

Retrieve the current user's profile information.

Response

{
  "success": true,
  "user": {
    "id": 1,
    "username": "johndoe",
    "email": "user@example.com",
    "created_at": "2025-01-01T00:00:00Z",
    "timezone": "Asia/Kolkata"
  }
}

Update User Profile

PUT /user/profile

Request Body

{
  "username": "newusername",
  "email": "newemail@example.com",
  "timezone": "America/New_York"
}

Events

Create, read, update, and delete events and tasks.

List Events

GET /events

Retrieve all events for the authenticated user.

Query Parameters

Parameter Type Description
start_date string Filter events from this date (YYYY-MM-DD)
end_date string Filter events until this date (YYYY-MM-DD)
limit integer Number of events to return (default: 50)

Response

{
  "success": true,
  "events": [
    {
      "id": 1,
      "title": "Team Meeting",
      "description": "Weekly team sync",
      "start_date": "2025-01-15",
      "start_time": "10:00:00",
      "end_time": "11:00:00",
      "category": "work",
      "color": "#3b82f6",
      "created_at": "2025-01-01T00:00:00Z"
    }
  ],
  "total": 1
}

POST /events

Request Body

{

  "start_date": "2025-01-15",

PUT /events/{id}

Delete Event

DELETE /events/{id}

Error Handling

Understanding API error responses and status codes.

HTTP Status Codes

200 OK - Request successful
201 Created - Resource created successfully
400 Bad Request - Invalid request data
401 Unauthorized - Authentication required
404 Not Found - Resource not found
500 Internal Server Error - Server error

Error Response Format

{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "The provided data is invalid",
    "details": {
      "title": "Title is required",
      "start_date": "Invalid date format"
    }
  }
}
